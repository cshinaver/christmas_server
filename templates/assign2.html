Objects, objects, and more objects

As you probably already noticed, Objective-C has its own types for almost every C type. The important thing to remember is that each of these ObjC types is an object. It is also an instance of a class (instance and object are essentially interchangable). Each Objective-C object has its own methods that can be very useful. For example, to get the number of elements in an NSString, you can use the -length method. (- sign in front of a method means it is an instance method, meaning you use it on an instance of a class. + in front of a method means you use it on the class itself. Good explanation here.
NSString *myString = @"Fancy Feast is my friend";
NSUinteger *lenStr = [myString length];
Note: Objective-C is smart about it's classes, meaning you can write a for loop that requires an int as a bound, but use the return value of type NSUinteger.
E.g:
for (int i = 0; i < [myString length], i++) // Note the initialization of i in the statement and its type
{
    NSLog(@"%C", [myString characterAtIndex:i]); // -characterAtIndex returns a unichar, so %C, not %c
    // Objective-C strings are not C-strings i.e. myString[i] does not work
}

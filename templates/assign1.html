sure everyone undersands object orientation and objective-C in general.

Objective-C uses the NSLog() command to display output on the console. Use this instead of printf statements when you want to log something.
The NSLog() command takes a string argument. This is not a C-String. It is an Objective-C string. Similarly to C++, Objective-C has its own implemented string class. Objective-C strings, henceforth referred to as NSStrings, are declared like so:
NSString *myString = @"Banana factory";

Note the asterick, @ sign and semicolon.

COLLEEEEEEEEEEEEEEEEN

Every line does end with semicolon in ObjC. 'Nuff said.
NSStrings require @ to be in front of double quotes when setting them. This is how Objc specifies the difference between a C-String and an NSString.
The asterisk is interesting. Basically, Objective-C loves pointers. Like loves them. It's ridiculous. Any Objective-C types (all objects, most begin with NS) must be created using a pointer.
If I wanted to define an Objective-C array, I would have to do it this way:
NSArray *myArray = [NSArray arrayWithObjects:@"I ", @"love ", [NSNumber numberWithInt:5], @" dogs", nil];
// nil is the sentinal value for the method arrayWithObjects

COLLEEEEEEEEEEEEEEEEN

Sidenote: NSArrays can store any type.
